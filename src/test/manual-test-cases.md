# Ручные тест-кейсы DaData

## UI-1. Главная страница отображает основной заголовок и CTA-кнопки

Предусловия: пользователь не авторизован.

Шаги:
1. Открыть `https://dadata.ru/`.
2. Проверить основной заголовок на первом экране.
3. Проверить наличие кнопок `Подключиться` и `Присоединиться`.

Ожидаемый результат: отображается заголовок `Наводим порядок в данных`, обе кнопки видимы и доступны.

## UI-2. Кнопка входа открывает форму авторизации

Предусловия: пользователь не авторизован.

Шаги:
1. Открыть `https://dadata.ru/`.
2. Нажать кнопку `Войти`.
3. Проверить поля email и пароль.
4. Проверить кнопку отправки формы.

Ожидаемый результат: открывается форма входа с полями email, пароль и кнопкой `Войти в систему`.

## UI-3. Кнопка подключения открывает форму регистрации

Предусловия: пользователь не авторизован.

Шаги:
1. Открыть `https://dadata.ru/`.
2. Нажать кнопку `Подключиться`.
3. Проверить поля имени, email и пароля.
4. Заполнить форму валидными тестовыми данными.

Ожидаемый результат: форма регистрации отображается, поля доступны для ввода.

## UI-4. Меню продуктов содержит основные продукты DaData

Предусловия: пользователь находится на главной странице.

Шаги:
1. Открыть меню `Продукты`.
2. Проверить список продуктовых пунктов.

Ожидаемый результат: в меню отображаются основные продукты, включая `Подсказки`, `Стандартизация`, `Организация по ИНН`, `Маскировщик`.

## UI-5. Переход в блог и открытие hot-статьи

Предусловия: пользователь находится на главной странице.

Шаги:
1. Нажать пункт меню `Блог`.
2. Проверить заголовок страницы блога.
3. Открыть hot-статью.
4. Проверить наличие разделов статьи.

Ожидаемый результат: открывается страница блога, затем статья с разделами `О компании`, `Проблема`, `Решение`, `Результат`.

## API-1. Очистка ФИО через Cleaner API

Предусловия: есть валидные `dadata.api.token` и `dadata.api.secret`.

Шаги:
1. Отправить POST-запрос `/api/v1/clean/NAME` с телом `["иванов иван иванович"]`.
2. Проверить статус ответа.
3. Проверить нормализованное ФИО и разобранные части.

Ожидаемый результат: статус 200, результат `Иванов Иван Иванович`, `qc = 0`.

## API-2. Очистка телефона через Cleaner API

Предусловия: есть валидные `dadata.api.token` и `dadata.api.secret`.

Шаги:
1. Отправить POST-запрос `/api/v1/clean/PHONE` с телом `["8 912 345-67-89"]`.
2. Проверить статус ответа.
3. Проверить формат телефона и код страны.

Ожидаемый результат: статус 200, телефон начинается с `+7`, `country_code = 7`, `qc = 0`.

## API-3. Подсказки адресов через Suggestions API

Предусловия: есть валидный `dadata.api.token`.

Шаги:
1. Отправить POST-запрос `/api/4_1/rs/suggest/address` с query `москва сухонская 11`.
2. Проверить статус ответа.
3. Проверить список подсказок и данные первой подсказки.

Ожидаемый результат: статус 200, список подсказок не пустой, первая подсказка относится к Москве.

## API-4. Подсказки email через Suggestions API

Предусловия: есть валидный `dadata.api.token`.

Шаги:
1. Отправить POST-запрос `/api/4_1/rs/suggest/email` с query `test.user@gm`.
2. Проверить статус ответа.
3. Проверить список подсказок.

Ожидаемый результат: статус 200, список подсказок не пустой, значение первой подсказки начинается с `test.user@`.
